# Bill Renamer - Agent Notes

## Project Overview
OCR-based invoice/bill auto-renamer. Extracts Invoice Number, Date, and Company Name from scanned invoices using PaddleOCR, then renames files.

**Stack**: Python 3.11, PaddleOCR (ONNX Runtime), CPU-only, offline
**Model**: PP-OCRv6_small (det + rec)
**Config**: `config.json` — `input_folder`, `jpeg_quality: 100`, `max_dimension: 0`

## Architecture
```
main.py          → entry point, orchestrates OCR → extract → rename
src/ocr.py       → PaddleOCR wrapper, returns (texts, boxes, img_width, img_height)
src/extractor.py → regex-based extraction for invoice#, date, company name
src/renamer.py   → filename generation and rename logic
src/utils.py     → config loading, image file discovery
src/logger.py    → rename logging to markdown
```

## Extraction Logic (`src/extractor.py`)

### Invoice Number (`extract_invoice_number`)
- Searches for "Invoice No" / "Bill No" keywords, extracts adjacent alphanumeric value
- Falls back to standalone 3+ digit numbers
- Returns `"UnknownInvoice"` if not found

### Date (`extract_date`)
- Regex for DD/MM/YYYY, DD-MM-YYYY, YYYY-MM-DD formats
- Falls back to `dateparser` library
- Returns `"UnknownDate"` if not found

### Company Name (`extract_name`) — **Most Complex**
The supplier name extraction uses a multi-stage filtering pipeline:

#### Stage 1: Basic Filters (reject candidate if...)
- Text < 2 chars
- Contains Chinese/CJK characters
- All digits/spaces/punctuation (regex `^[\d\s\-/\.:]+$`)
- Matches skip_pattern regex (200+ invoice/label keywords)
- Matches _product_re (product/goods names)
- Matches user-configured skip_words
- Text > max_length (30 chars)
- More than max_commas (2) commas
- Contains dot-space pattern (`\w\.\s*\w`) — catches "Disc. %", "Incl. of Tax"
- After stripping parentheses, text < 3 chars
- More than max_numbers (2) digits in cleaned text

#### Stage 2: Spatial Filter
- `top_half = img_height / 2` — only consider text boxes in upper half of image
- Originally was `top_third = img_height / 3` but was too aggressive (cut off valid supplier names on some layouts)
- **Note**: Originally computed but never used — bug fixed in this session

#### Stage 3: Scoring & Selection
- Score = bounding box height in pixels
- Candidates sorted by score descending (tallest box wins)
- Confidence threshold: `_confidence_threshold = 20` (reject if best score < 20px)
- **Known limitation**: Box height is not always correlated with supplier name importance

## Changes Made This Session

### 1. Parenthetical Regex Fix
**File**: `extractor.py:238`
- **Before**: `re.sub(r'\([\w\s\-/&]+\)', '', clean)` — period in `(Incl. of Tax)` blocked match
- **After**: `re.sub(r'\([^)]*\)', '', clean)` — matches any parentheses content
- **Impact**: `(Incl. of Tax)` now correctly stripped, preventing it from winning

### 2. Confidence Threshold
**File**: `extractor.py:207`
- **Before**: `_confidence_threshold = 50` — rejected valid 41px-tall company names
- **After**: `_confidence_threshold = 20` — more permissive
- **Impact**: `TRANSPARENT SYSTEMS` (h=41) no longer killed by threshold

### 3. Dot-in-Middle Filter
**File**: `extractor.py:235`
- **Added**: `if re.search(r'\w\.\s*\w', clean): return None`
- **Catches**: `(Incl. of Tax)`, `Disc. %`, `A/c No.`, `E-Mail`, etc.
- **Impact**: Filters labels/descriptions with abbreviations

### 4. Top-Third Bug Fix
**File**: `extractor.py:250-257`
- **Bug**: `top_third = img_height / 3` was computed but never used in candidate loop
- **Fix**: Added `if box_top > top_third: continue` filter
- **Then changed to**: `top_half = img_height / 2` (less aggressive)
- **Impact**: Prevents product names from bottom of image from winning

### 5. Product Name Filter
**File**: `extractor.py:195-203`
- **Added**: `_product_re` regex for product/goods names (processor, motherboard, cabinet, etc.)
- **Why**: `\bmotherboard\b` skip pattern didn't match `ASUSMOTHERBOARD` (no word boundary in compound words)
- **Impact**: `ASUSMOTHERBOARD`, `CI5 PROCESSOR` correctly rejected

### 6. Uppercase Boost Removed
**File**: `extractor.py:246-248`
- **Before**: `if clean.isupper(): score *= 1.1`
- **Removed**: Was unfairly favoring ALL-CAPS product names over mixed-case supplier names
- **Impact**: More neutral scoring

### 7. Skip Pattern Expansion (~200+ patterns)
**File**: `extractor.py:102-192`
- **Added categories**:
  - Invoice labels: invoice, bill, eway, ack, irn, etc.
  - Financial terms: cgst, sgst, igst, cess, round off, etc.
  - Indian cities: Nagpur, Mumbai, Delhi, Chennai, etc. (~20 cities)
  - Indian states: Maharashtra, Gujarat, Rajasthan, etc. (~30 states/UTs)
  - Address words: Road, Street, Nagar, Park, Building, C/o, etc.
  - E-Way Bill labels: Approx Distance, Supply Type, Generated By, etc.
  - Table headers: HSN, SAC, Description, Quantity, Rate, Amount, etc.
  - Product units: NOS, MTR, KG, LTR, PC, PCS, etc.
  - Company suffixes: Pvt, Ltd, LLP, Co, Company, etc.
  - Action labels: Sold by, Manufactured by, Marketed by, etc.

### 8. Chinese Character Filter
**File**: `extractor.py:205`
- **Added**: `_chinese_re` regex for CJK characters
- **Impact**: Rejects any text with Chinese characters

## Known Limitations

### 1. Box Height Heuristic
The "tallest bounding box wins" approach is fundamentally unreliable:
- Table headers (Disc. %, NOS, 1,661) often have taller boxes than supplier names
- Supplier names on some layouts are in smaller text
- Every new skip pattern filters one false positive but lets another through

### 2. Skip Pattern Arms Race
Current skip_pattern has 200+ patterns and still misses cases:
- `Carting on Sales` — generic accounting term, not a supplier
- `GRAPHICS_CARD` — product name, should be filtered
- New false positives appear as skip patterns grow

### 3. Spatial Filter Trade-off
- `top_third`: Cuts off valid supplier names on some layouts (IN_Dated, IN_Mode)
- `top_half`: Too permissive, lets false positives from bottom compete
- `no filter`: Worst — product names from anywhere win

### 4. E-Way Bill Format
E-Way Bills (IN_Mode, IN_Dated) have different structure:
- Supplier info is in the middle/bottom of the page
- Lots of tall labels above the supplier section
- Invoice number format differs from standard tax invoices

## Future Improvements (Not Yet Implemented)

### Option A: Known Supplier List (config.json)
```json
{
  "known_suppliers": ["TRANSPARENT SYSTEMS", "OM TEXTILES", "V DECOR"]
}
```
- Match OCR text against known list using fuzzy matching
- Most reliable but requires manual configuration
- Estimated: 2-3 hours to implement, ~85% accuracy

### Option B: LayoutLMv3 (ML-based)
- Pre-trained document understanding model
- Requires training data (50+ labeled invoices per supplier)
- Estimated: 15-30 hours (50% is data annotation)
- Best long-term solution but heavy upfront cost

### Option C: Hybrid Approach
- Use known supplier list for known invoices
- Fall back to heuristic extraction for new suppliers
- Estimated: 3-4 hours to implement
